# Отчет по лабораторной работе 9

## Тема работы

Исследование алгоритма хеширования `MD5`: реализация алгоритма, оценка лавинного эффекта, поиск коллизий на случайных данных и измерение скорости вычисления.

## Метод / модель

В работе использована собственная реализация MD5 (блоковая обработка по 512 бит, 4 регистра состояния, 64 шага преобразования).

Мини-фрагмент реализации:

```java
for (int i = 0; i < 64; i++) {
    int f;
    int g;
    if (i < 16) {
        f = (b & c) | ((~b) & d);
        g = i;
    } else if (i < 32) {
        f = (d & b) | ((~d) & c);
        g = (5 * i + 1) % 16;
    } else if (i < 48) {
        f = b ^ c ^ d;
        g = (3 * i + 5) % 16;
    } else {
        f = c ^ (b | (~d));
        g = (7 * i) % 16;
    }
    int sum = a + f + TABLE[i] + words[g];
    b = b + Integer.rotateLeft(sum, SHIFT[i]);
}
```

## Шаг 1. Пары строк с отличиями 1,2,4,8,16 символов

### Что делалось

- Для каждого значения отличий сгенерировано 1000 пар строк длиной 128.
- Для каждой пары вычислены MD5-хеши.
- Для каждой пары найден максимум длины одинаковой **подстроки** в двух хешах.
- Для каждого значения отличий взят максимум среди 1000 пар.

Фрагмент кода:

```java
for (int i = 0; i < 1000; i++) {
    String first = randomString(128);
    String second = mutateString(first, diff);
    String hashA = Md5Hasher.hashHex(first);
    String hashB = Md5Hasher.hashHex(second);
    int longest = longestCommonSubstring(hashA, hashB);
    globalMax = Math.max(globalMax, longest);
}
```

### Результат

- `diff=1 -> max=5`
- `diff=2 -> max=5`
- `diff=4 -> max=4`
- `diff=8 -> max=5`
- `diff=16 -> max=4`

График: `lab9_results/chart_diff_vs_common.png`

Краткий вывод: общие последовательности в хешах короткие (4-5 символов), что согласуется с ожидаемым лавинным эффектом.

## Шаг 2. Генерация N=10^i (i=2..6) и поиск одинаковых хешей

### Что делалось

- Для каждого `N` генерировались случайные строки длины 256.
- Вычислялся MD5 каждой строки.
- В `Map<hash, count>` считалась частота каждого хеша.
- Повтор считался коллизией, если частота становилась больше 1.

Фрагмент кода:

```java
Map<String, Integer> frequency = new LinkedHashMap<>(n * 2);
int duplicateCount = 0;
for (int i = 0; i < n; i++) {
    String hash = Md5Hasher.hashHex(randomString(256));
    int next = frequency.getOrDefault(hash, 0) + 1;
    frequency.put(hash, next);
    if (next > 1) {
        duplicateCount++;
    }
}
```

### Результат (таблично)

- `100 -> 0 коллизий`
- `1000 -> 0 коллизий`
- `10000 -> 0 коллизий`
- `100000 -> 0 коллизий`
- `1000000 -> 0 коллизий`

Подробно: `lab9_results/collision_experiment.csv` и `lab9_results/summary.md`.

Краткий вывод: в рамках данного объема случайной выборки коллизии не обнаружены.

## Шаг 3. Скорость вычисления MD5 от размера входа

### Что делалось

- Для размеров `64, 128, 256, 512, 1024, 2048, 4096, 8192` проведено по 1000 измерений.
- На каждом измерении генерировалась новая случайная строка.
- Считалось среднее время в наносекундах и скорость в хешах/с.

Фрагмент кода:

```java
for (int i = 0; i < 1000; i++) {
    String data = randomString(size);
    long start = System.nanoTime();
    Md5Hasher.hashHex(data);
    totalNs += (System.nanoTime() - start);
}
double avgNs = totalNs / 1000.0;
double hashesPerSecond = 1_000_000_000.0 / avgNs;
```

### Результат

- На малых размерах (`64..512`) скорость выше.
- При росте размера до `8192` скорость падает до ~`13373` хеш/с.

График: `lab9_results/chart_size_vs_speed.png`

Краткий вывод: наблюдается ожидаемая зависимость — чем больше вход, тем больше среднее время хеширования и ниже пропускная способность.

## Общий вывод по работе

В ходе лабораторной работы была полностью реализована и проверена собственная версия алгоритма MD5: от этапа дополнения сообщения и разбиения на блоки до 64-шаговой раундовой обработки и формирования итогового 128-битного хеша. Отдельно важно, что эксперименты по всем пунктам задания автоматизированы, поэтому результаты воспроизводимы и могут быть пересчитаны в любой момент на том же коде.

Анализ первого эксперимента показывает устойчивый лавинный эффект: даже при изменении входной строки на 1-2 символа максимальные общие последовательности в парах хешей остаются короткими (порядка 4-5 символов). График `chart_diff_vs_common.png` подтверждает, что заметной тенденции к увеличению совпадающих фрагментов нет — хеш быстро «теряет» структурное сходство с исходным текстом.

Во втором эксперименте, при генерации от `10^2` до `10^6` случайных строк длиной 256 символов, совпадающих хешей не обнаружено. Такой результат ожидаем для данного объема выборки и 128-битного пространства значений MD5: отсутствие коллизий в эксперименте не доказывает их невозможность в общем случае, но подтверждает хорошее распределение хеша на случайных входах практического масштаба в рамках лабораторной.

Третий эксперимент демонстрирует типичное поведение производительности: с ростом длины входной строки среднее время вычисления увеличивается, а скорость в хешах/сек снижается. На графике `chart_size_vs_speed.png` это видно как нисходящий тренд пропускной способности при переходе от коротких строк к более длинным (до 8192 символов). Небольшие локальные колебания между соседними точками объясняются влиянием JVM и системных факторов измерения времени, но общая зависимость сохраняется.

Итог: поставленные цели лабораторной работы достигнуты полностью. Реализация MD5 корректно работает, экспериментальная часть выполнена по всем требованиям задания, графики и таблицы согласуются с теоретическими ожиданиями для криптографической хеш-функции.
