# Теоретическая часть по хешированию и MD5

## 1. Что такое хеш-функция

Хеш-функция — это алгоритм, который преобразует входные данные произвольной длины в строку фиксированной длины (хеш).
Для криптографических хеш-функций важны свойства:

- **Детерминированность**: одинаковый ввод всегда дает одинаковый хеш.
- **Лавинный эффект**: малое изменение входа заметно меняет выход.
- **Стойкость к предобразу**: по хешу сложно восстановить исходные данные.
- **Стойкость ко второму предобразу**: сложно найти другой ввод с тем же хешем.
- **Коллизионная стойкость**: сложно найти две разные строки с одинаковым хешем.

Хеш не является шифрованием: исходный текст по нему обратно не восстанавливается.

## 2. Где применяются хеши

- контроль целостности файлов;
- индексирование и быстрый поиск;
- хранение паролей (в современных системах — через специальные медленные алгоритмы типа bcrypt/Argon2, а не MD5);
- контроль неизменности сообщений;
- дедупликация данных.

## 3. Почему в лабораторной выбран MD5

По варианту задания требуется реализовать `MD5`.  
MD5 исторически очень важен и хорошо подходит для учебной цели:

- компактная структура (блоки 512 бит, 4 регистра состояния);
- понятные раунды с побитовыми операциями;
- удобно исследовать лавинный эффект и вероятность коллизий на случайных данных.

Важно: MD5 **не рекомендуется для криптографической защиты** в реальных системах из-за известных атак на коллизии.

## 4. Математическая модель MD5 (кратко)

1. Сообщение дополняется:
   - бит `1`,
   - затем нули до длины `≡ 448 (mod 512)`,
   - затем 64-битная длина исходного сообщения (в little-endian).
2. Данные делятся на блоки по 512 бит (16 слов по 32 бита).
3. Состояние инициализируется четырьмя 32-битными регистрами:
   - `A = 0x67452301`
   - `B = 0xEFCDAB89`
   - `C = 0x98BADCFE`
   - `D = 0x10325476`
4. Каждый блок проходит 64 шага (4 раунда по 16 шагов), где используются:
   - нелинейные функции `F, G, H, I`,
   - таблица констант `T[i] = floor(abs(sin(i+1)) * 2^32)`,
   - циклические сдвиги влево.
5. Результат — конкатенация `A, B, C, D` в little-endian, 128 бит (32 hex-символа).

## 5. Что реализовано в проекте

- `Md5Hasher` — собственная реализация MD5 без `MessageDigest`.
- `HashLabRunner` — автоматический прогон всех трех экспериментов из задания.
- `HashLabCharts` — построение двух графиков (п.1 и п.3).
- `HashLabMain` — точка входа:
  - обычный режим: хеш одной строки;
  - режим `--run-lab`: полный прогон лабораторной.

## 6. На что смотреть при анализе результатов

- Для п.1: при малых изменениях входа общие последовательности в хешах должны оставаться короткими.
- Для п.2: при `N` до `10^6` на случайных данных коллизии MD5 могут не встретиться (что согласуется с оценкой по дню рождения для 128-битного пространства).
- Для п.3: по мере роста размера входа скорость (хеш/с) падает, среднее время (нс) растет.
