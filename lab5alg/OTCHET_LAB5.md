## 1. Описание метода и модели

### 1.1. Математическая модель

В работе рассматривается неориентированный взвешенный связный граф \(G = (V, E)\), где \(V\) — множество вершин, вершины нумеруются с 0 до \(N-1\)), а каждое ребро \(\{i, j\}\) имеет **целый вес** \(w_{ij} = w_{ji}\) из диапазона **от 1 до 20** включительно. Отсутствие ребра между вершинами \(i\) и \(j\) задаётся нулём в матрице смежности.

**Степень вершины** — число рёбер, инцидентных этой вершине. По условию лабораторной для каждого допустимого (N) задана **минимальная** степень каждой вершины: при \(N = 10\) не меньше 3, при \(N = 20\) — 4, при \(N = 50\) — 10, при \(N = 100\) — 20. Фактическое число соседей может быть больше минимума за счёт случайного добавления рёбер.

**Связность** означает, что для любых двух вершин существует путь по рёбрам графа.

**Кратчайший путь** между вершинами \(s\) и \(t\) — путь, на котором **сумма весов рёбер** минимальна среди всех путей от \(s\) к \(t\). В отчёте и программе основным результатом для варианта 1 является **матрица длин** кратчайших путей между всеми упорядоченными парами \((s, t)\).

### 1.2. Способ задания графа в программе

Граф хранится в виде **квадратной симметричной матрицы весов**: элемент \((i, j)\) положителен, если ребро есть, и равен нулю, если ребра нет; на главной диагонали нули (петель нет). Класс `WeightedGraph` инкапсулирует эту матрицу и проверяет корректность данных при создании.

```java
// Фрагмент: конструктор WeightedGraph — квадратность, симметрия, нулевая диагональ
public WeightedGraph(int[][] weights) {
    int n = weights.length;
    for (int[] row : weights) {
        if (row.length != n) {
            throw new IllegalArgumentException("Матрица должна быть квадратной");
        }
    }
    this.weights = deepCopy(weights);
    for (int i = 0; i < n; i++) {
        if (this.weights[i][i] != 0) {
            throw new IllegalArgumentException("Диагональ должна быть нулевой");
        }
        for (int j = i + 1; j < n; j++) {
            if (this.weights[i][j] != this.weights[j][i]) {
                throw new IllegalArgumentException("Матрица должна быть симметричной");
            }
        }
    }
}
```

### 1.3. Метод решения задачи «все пары»

Для графа с **неотрицательными** весами кратчайшие пути от одной стартовой вершины ко всем остальным находятся **алгоритмом Дейкстры** (жадный перебор с приоритетной очередью по текущей оценке расстояния).

Чтобы получить расстояния **между всеми парами**, метод варианта 1 реализуется так: для **каждой** вершины \(s\) один раз запускается Дейкстра со стартом в \(s\); строка \(s\) итоговой матрицы совпадает с вектором расстояний от \(s\).

### 2.1. Генерация графа с заданными ограничениями

Реализован класс `ConnectedWeightedGraphGenerator`. Минимальные степени задаются в соответствии с таблицей из лабораторной работы:

```java
public static int requiredMinDegree(int n) {
    return switch (n) {
        case 10 -> 3;
        case 20 -> 4;
        case 50 -> 10;
        case 100 -> 20;
        default -> throw new IllegalArgumentException("Допустимы только N из {10, 20, 50, 100}: " + n);
    };
}
```

Алгоритм генерации:

1. Строится **случайное остовное дерево** на всех \(N\) вершинах: каждое новое ребро получает случайный вес из \([1, 20]\). Это гарантирует **связность** графа.
2. Пока у какой-либо вершины степень меньше требуемого минимума, добавляется новое ребро к случайной вершине, с которой ещё нет ребра, с тем же распределением весов.
3. Выполняется проверка **связности** (обход в ширину) и проверка выполнения **минимальной степени** для всех вершин. При необходимости генерация повторяется.

Фрагмент внешнего цикла генерации и вызова проверок:

```java
public static WeightedGraph generate(int n, Random random) {
    int minDeg = requiredMinDegree(n);
    if (minDeg > n - 1) {
        throw new IllegalArgumentException("Невозможно иметь степень >= " + minDeg + " при N=" + n);
    }
    for (int attempt = 0; attempt < 500; attempt++) {
        int[][] m = tryBuild(n, minDeg, random);
        if (m != null && isConnected(m) && minDegreeOk(m, minDeg)) {
            return new WeightedGraph(m);
        }
    }
    throw new IllegalStateException("Не удалось сгенерировать граф за отведённые попытки: N=" + n);
}
```

Таким образом выполняются требования: \(N \in \{10, 20, 50, 100\}\), веса \(1 \ldots 20\), граф связный, степени не ниже заданных порогов.

### 2.3. Задание 3, вариант 1: все пары, алгоритм Дейкстры

Класс `DijkstraAllPairs` строит матрицу `dist`, где `dist[s][t]` — длина кратчайшего пути из \(s\) в \(t\). Для каждого \(s\) вызывается одна Дейкстра по матрице весов:

```java
public static int[][] shortestPathLengths(WeightedGraph graph) {
    int[][] w = graph.adjacencyMatrix();
    int n = w.length;
    int[][] all = new int[n][n];
    for (int s = 0; s < n; s++) {
        all[s] = dijkstraFromSource(w, s);
    }
    return all;
}
```

Ядро Дейкстры: приоритетная очередь по текущей дистанции, пропуск устаревших записей в очереди, релаксация рёбер:

```java
public static int[] dijkstraFromSource(int[][] w, int start) {
    int n = w.length;
    int[] dist = new int[n];
    Arrays.fill(dist, Integer.MAX_VALUE);
    dist[start] = 0;
    PriorityQueue<int[]> pq = new PriorityQueue<>(Comparator.comparingInt(a -> a[1]));
    pq.add(new int[]{start, 0});
    while (!pq.isEmpty()) {
        int[] cur = pq.poll();
        int u = cur[0];
        int d = cur[1];
        if (d > dist[u]) {
            continue;
        }
        for (int v = 0; v < n; v++) {
            if (u == v) continue;
            int ww = w[u][v];
            if (ww <= 0) continue;
            long nd = (long) d + ww;
            if (nd < dist[v]) {
                dist[v] = (int) nd;
                pq.add(new int[]{v, dist[v]});
            }
        }
    }
    return dist;
}
```

Дополнительно выполняется проверка: для связного неориентированного графа все расстояния конечны, матрица симметрична, на диагонали нули (`verifyAllPairsUndirected` в `Main`).

Результат для отчёта выводится в виде полной матрицы расстояний при \(N \le 20\) и фрагмента при больших \(N\), при этом полная матрица в памяти вычисляется всегда.

Граф задаётся и демонстрируется в виде **матрицы смежности**.

Замерялось только время этапа «все пары» по семи прогонам на каждое (N) берётся среднее и строится график. На графике видно, что при росте (N) среднее время увеличивается: это ожидаемо, потому что запусков Дейкстры становится больше (их (N)), а каждый запуск обрабатывает граф большего размера и с большим числом рёбер при тех же ограничениях на степень. Итог лабораторной — не только «написали алгоритм», а воспроизводимый конвейер: генерация по условию → матрица смежности → матрица расстояний → измерение и наглядная зависимость времени от (N).
