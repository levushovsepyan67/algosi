# Лабораторная работа №5: теория и защита (вариант 1)

Материал для отчёта и устной защиты. Соответствует коду в проекте `lab5alg` (пакеты `usaskoviy.graph`, `usaskoviy.algo`, `usaskoviy.analysis`, класс `Main`).

---

## 1. Цель работы по условию

1. Сгенерировать **связный неориентированный взвешенный** граф с параметрами:
   - число вершин **N ∈ {10, 20, 50, 100}**;
   - у каждой вершины **не меньше** заданного числа соседей (минимальная степень);
   - вес каждого ребра — **целое случайное число от 1 до 20**;
   - граф **связный** (из любой вершины можно добраться до любой).

2. Вывести граф в виде **матрицы смежности** (в коде — для первой пробы каждого N, чтобы не дублировать огромный вывод).

3. **Вариант 1:** найти **кратчайшие пути между всеми парами вершин** (в реализации вычисляются **длины** этих путей) с помощью **алгоритма Дейкстры**.

4. Для каждого N провести **от 5 до 10** экспериментов, **измерить время**, построить **график**: по оси X — **N**, по оси Y — **время**.

---

## 2. Основные понятия

### 2.1. Взвешенный граф

**Граф** G = (V, E): множество **вершин** V и **рёбер** E.

**Взвешенный** граф: каждому ребру сопоставлено **неотрицательное число** — **вес** (здесь целое 1…20).

**Неориентированное** ребро: переход возможен в обе стороны; матрица смежности **симметрична**.

**Длина пути** — сумма весов рёбер на пути (не число рёбер).

**Кратчайший путь** между u и v — путь с **минимальной** суммой весов.

### 2.2. Матрица смежности с весами

Квадратная матрица размера N×N. Элемент a[i][j]:

- **0** — ребра между i и j нет (в печати можно оставить пустую клетку);
- **положительное число** — вес ребра между i и j.

Для неориентированного графа a[i][j] = a[j][i]. На диагонали обычно **0** (петель нет).

### 2.3. Степень вершины

**Степень** вершины — число инцидентных ей рёбер. В матричной форме — сколько ненулевых элементов в строке (кроме диагонали).

В задании задана **нижняя граница** степени в зависимости от N (таблица в условии).

### 2.4. Связный граф

Граф **связный**, если между любыми двумя вершинами существует путь по рёбрам.

В программе связность обеспечивается построением **остовного дерева** (N−1 ребро, все вершины в одной компоненте), затем добавляются дополнительные рёбра для степеней.

---

## 3. Идея генератора (как в `ConnectedWeightedGraphGenerator`)

1. **Случайное дерево** на всех вершинах: вершины 1…N−1 в случайном порядке присоединяются к случайной уже включённой вершине; каждому новому ребру задаётся случайный вес 1…20. После этого граф **связный**.

2. Пока у какой-то вершины степень **меньше** требуемого минимума, добавляется **новое** ребро к случайной вершине, с которой ещё нет ребра, снова со случайным весом 1…20.

3. Проверки: **связность** (обход в ширину), **минимальная степень** у всех вершин.

Если за ограниченное число попыток построить не удалось, выполняется новая попытка с начала.

---

## 4. Алгоритм Дейкстры

**Назначение:** из одной **стартовой** вершины s найти кратчайшие расстояния до всех остальных при **неотрицательных** весах.

**Идея:** поддерживать оценку dist[v] — текущее лучшее известное расстояние от s до v. На каждом шаге брать вершину u с **минимальным** dist среди незафиксированных, «релаксировать» рёбра (u, w): если dist[u] + вес(u, w) < dist[w], обновить dist[w].

**Структура данных:** **очередь с приоритетом** (min-heap) по текущему dist у пар (вершина, расстояние). Устаревшие записи в куче отбрасываются, если извлечённое расстояние больше актуального dist[u].

**Сложность:** с бинарной кучей при разреженном графе часто пишут **O((V + E) log V)** на один запуск; при плотной матрице перебор всех v для каждого u даёт порядка **O(V² log V)** из-за кучи (в коде соседи перебираются по строке матрицы).

### 4.1. Все пары вершин (вариант 1)

Чтобы получить расстояния **между всеми парами**, запускают Дейкстру **N раз**: для каждого s = 0…N−1 старт — вершина s, в результате строка s матрицы ответа.

**Итоговая сложность** (порядок): **N × (один запуск Дейкстры)**. Альтернатива — алгоритм Флойда–Уоршелла O(V³) без кучи; в задании указана именно **Дейкстра**.

---

## 5. Эксперимент и график

- Для каждого N из {10, 20, 50, 100} генерируется **несколько** (в коде 7) разных случайных графов.
- Для каждого графа замеряется время выполнения **полного** блока «все пары через N запусков Дейкстры» (`System.nanoTime()`).
- Для точки на графике по оси Y берётся **среднее** время по этим прогонам (удобно для сглаживания шума измерений).
- Ось X — N; ось Y — время в **микросекундах** (наносекунды / 1000) для читаемости.

**Интерпретация:** при росте N растёт и число запусков Дейкстры (N), и размер графа на каждом запуске, поэтому время обычно растёт **быстрее линейно** относительно N.

## 7. Типичные вопросы на защите

**Почему Дейкстра работает только при неотрицательных весах?**  
При отрицательном ребре «жадный» выбор минимального dist может быть неверным; нужны другие алгоритмы (например, Беллман–Форд).

**Чем отличается кратчайший путь в невзвешенном и взвешенном графе?**  
В невзвешенном минимизируют число рёбер (BFS). Здесь минимизируют **сумму весов**.

**Зачем дерево в генераторе?**  
Чтобы **заранее** гарантировать одну связную компоненту на всех N вершинах.

**Почему на графике среднее, а не одна проба?**  
Снижает влияние случайного разброса времени на конкретном графе и соответствует здравому смыслу отчёта по серии экспериментов.
