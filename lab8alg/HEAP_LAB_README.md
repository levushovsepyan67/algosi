# Лабораторная: бинарная min-куча и биномиальная min-куча (вариант 2)

## Что реализовано

- `BinaryMinHeap` — бинарная min-куча на массиве.
- `BinomialMinHeap` — биномиальная min-куча (список корней, слияние с «переносом» по степеням).
- `HeapLabMain` — замеры по заданию и автоматический вывод результатов.
- `HeapCharts` — графики (JFreeChart).

## Условие эксперимента (как в методичке)

- Размеры: `N = 10^i`, `i = 3..7` (то есть `1000` … `10 000 000`).
- После заполнения кучи `N` элементами для каждой структуры замеряются **по 1000** операций:
  - `findMin` (просмотр минимума),
  - `deleteMin` (удаление минимума),
  - `insert` (вставка нового ключа).
- Считаются:
  - суммарное время серии из 1000 (в отчёте удобнее смотреть **среднее** = сумма / 1000),
  - **максимум** времени одной операции в серии,
  - дополнительно: **максимум средних по пакетам** по 25 операций (аналог «пика» при шуме `nanoTime`).

## Запуск

Точка входа: `usaskoviy.heaps.HeapLabMain`.

В IntelliJ: Run `HeapLabMain`.  
С Maven (если установлен): `mvn -q compile exec:java`.

После выполнения появится каталог `heap_lab_out/`:

- `heap_lab_metrics.csv` — все числа,
- `heap_avg_*.png`, `heap_max_*.png` — графики среднего и максимума для трёх операций.

## Примечание

Для `N = 10^7` прогон может занять много времени и памяти (особенно у биномиальной кучи из-за узловых объектов). При нехватке памяти временно уберите `7` из массива `POWERS` в `HeapLabMain`.
