**Тема:** исследование эффективности сортировки слиянием (вариант 2)

### 1. Описание алгоритма

Сортировка слиянием — это устойчивый сравнивающий алгоритм сортировки, который реализует принцип «разделяй и властвуй»: исходный массив рекурсивно делится на более мелкие части, а затем отсортированные части поочерёдно сливаются в один общий отсортированный массив.  
Алгоритм сортировки слиянием относится к классу алгоритмов «разделяй и властвуй».  
Основная идея работы такова:

1. Если подмассив состоит из одного элемента, он уже отсортирован.
2. Исходный массив рекурсивно делится на две примерно равные части.
3. Для каждой части рекурсивно выполняется сортировка тем же алгоритмом.
4. После этого две отсортированные половины **сливаются** в один отсортированный подмассив.

На стадии слияния одновременно просматриваются два отсортированных подмассива: из них по одному элементу выбирается меньший и записывается во временный буфер. Когда один из подмассивов закончился, оставшаяся часть второго просто дописывается в буфер. В конце содержимое буфера копируется обратно в исходный массив на тот же диапазон индексов.

Теоретические оценки:

- на каждом уровне рекурсии обрабатываются все n элементов, а число уровней пропорционально log₂n, поэтому время работы алгоритма оценивается как  
  T(n) = O(n · log n);
- для работы алгоритма требуется вспомогательный массив длины n, поэтому дополнительная память оценивается как  
  M(n) = O(n).

### 2. Описание выполнения задачи (код и реализация)

Программа реализована на языке Java и основана на структуре первой лабораторной работы (серийный запуск сортировки и построение графиков).

**2.1. Класс `MergeSort` (реализация алгоритма)**

- Метод `sort(double[] a)` создаёт один вспомогательный массив `temp` той же длины, что и исходный, и вызывает рекурсивную функцию `mergeSort` для всего диапазона индексов `[0; n-1]`.
- Вспомогательный внутренний класс `Counter` используется для сбора статистики:
  - `recursiveCalls` — сколько раз была вызвана рекурсивная функция `mergeSort`;
  - `maxDepth` — максимальная глубина рекурсии (уровень самого глубокого вызова);
  - `maxExtraBytes` — максимальный объём дополнительной памяти в байтах (размер массива `temp`).
- Метод `mergeSort`:
  - увеличивает счётчик вызовов и при необходимости обновляет текущую глубину рекурсии;
  - проверяет базу рекурсии: если `left >= right`, подмассив длины 1 или 0 не требует сортировки;
  - находит середину `mid` и рекурсивно сортирует левую часть `[left; mid]` и правую часть `[mid+1; right]`;
  - после сортировки обеих частей вызывает функцию `merge` для их слияния.
- Метод `merge` реализует сам процесс слияния двух соседних отсортированных подмассивов в буфер `temp` с последующим копированием результата обратно в исходный массив.

**2.2. Класс `MergeSortExperiment` (проведение эксперимента)**

- Задаёт массив размеров `SIZES = {1000, 2000, 4000, 8000, 16000, 32000, 64000, 128000}` и число запусков `RUNS_PER_SIZE = 20` для каждой серии.
- Для каждого размера:
  - генерирует 20 массивов случайных чисел с плавающей запятой, равномерно распределённых в интервале \([-1; 1]\);
  - для каждого массива измеряет время сортировки в наносекундах (с последующим переводом в миллисекунды);
  - получает от `MergeSort` статистику: глубину рекурсии, количество рекурсивных вызовов и дополнительную память;
  - сохраняет результаты в четыре массива (время, глубина, вызовы, память).
- После завершения серии вычисляются:
  - минимальное, максимальное и среднее время выполнения;
  - минимальная, максимальная и средняя глубина рекурсии;
  - минимальное, максимальное и среднее число рекурсивных вызовов;
  - минимальный, максимальный и средний объём дополнительной памяти.
- Все агрегированные данные по сериям выводятся в файл `results/merge_data.csv` (одна строка на размер массива) и кратко печатаются в консоль в виде таблицы.

### 3. Выводы и анализ графиков

Эксперимент выполнен для указанных восьми размеров массивов (от 1000 до 128000 элементов), по 20 запусков на каждый размер. Результаты всех запусков агрегированы и сохранены в файле `results/merge_data.csv`. Для размеров 1000–128000 наблюдается монотонный рост среднего и наихудшего времени сортировки при увеличении n. Глубина рекурсии для каждого размера близка к log₂n, количество рекурсивных вызовов примерно равно 2n − 1, а дополнительная память линейно растёт вместе с размером массива (один вспомогательный массив длины n). Построены все пять графиков в папке `results/`, что позволяет наглядно проанализировать зависимости времени, глубины рекурсии, числа вызовов и объёма памяти от числа элементов в массиве.

1. **Сложность по времени.**  
   По экспериментальным данным среднее время сортировки растёт быстрее, чем линейно, но значительно медленнее, чем квадратично. Наложение теоретической кривой O(c · n · log n) на график наихудших времен показывает хорошее совпадение формы кривой с экспериментальными точками, что подтверждает оценку сложности сортировки слиянием O(n · log n).

2. **Глубина рекурсии.**  
   Глубина рекурсии увеличивается очень медленно: для рассматриваемого диапазона размеров она меняется примерно от 11 до 18 уровней. Это хорошо согласуется с теоретической оценкой примерно равной ceil(log₂n), поскольку при каждом рекурсивном шаге массив делится пополам.

3. **Количество рекурсивных вызовов.**  
   Число вызовов рекурсивной функции растёт практически линейно от размера массива и близко к величине \(2n - 1\), что соответствует полному бинарному дереву разбиения задачи: каждая вершина дерева — один вызов `mergeSort`.

4. **Дополнительная память.**  
   Объём дополнительной памяти пропорционален размеру массива и определяется размером вспомогательного массива `temp`. Это подтверждает теоретическую оценку O(n) по памяти: при удвоении размера массива объём выделяемой памяти также примерно удваивается.

5. **Общая характеристика алгоритма.**  
   По результатам лабораторной работы можно сделать следующие выводы. По таблице и графикам видно, что по мере увеличения размера массива время работы и число рекурсивных вызовов растут заметно быстрее, чем линейно, но существенно медленнее, чем квадратично, что соответствует теоретической оценке сложности порядка O(n · log n). Зависимость времени выполнения от размера массива хорошо согласуется с теоретической кривой O(c · n · log n), построенной на графике. Эксперимент показал, что для небольших массивов сортировка слиянием работает очень быстро и остаётся практически «незаметной», а при переходе к большим размерам рост времени и потребление памяти становятся ощутимыми, но всё равно значительно лучше, чем у квадратичных алгоритмов сортировки.
