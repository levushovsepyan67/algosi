## Лабораторная работа №3 по алгоритмам  
**Тема:** реализация стека двумя способами и сравнение их работы

---

### 1. Постановка задачи

В рамках лабораторной работы необходимо изучить структуру данных стек и реализовать её двумя способами: на основе динамического массива и на основе односвязного списка. Для обеих реализаций нужно обеспечить операции добавления элемента на вершину и извлечения элемента с вершины, проверку стека на пустоту, подсчёт количества элементов и возможность обхода содержимого с помощью собственного итератора. Кроме того, требуется разработать набор тестов, демонстрирующих работу стека на целых числах, строках и объектах с датой рождения, проверить корректность вычисления агрегатных характеристик (сумма, среднее, минимум, максимум), показать пример инвертирования упорядоченной последовательности только с использованием операций стека, а также экспериментально сравнить две реализации между собой по скорости выполнения операций и по использованию памяти на коллекции из 10000 элементов.

---

### 2. Описание алгоритмов и структур данных

#### 2.1. Интерфейс стека

Для унификации реализаций введён интерфейс `SimpleStack<T>` (файл `SimpleStack.java`).  
Он задаёт стандартный набор операций:

- `push(T value)` — положить элемент на вершину стека;
- `pop()` — снять элемент с вершины;
- `peek()` — посмотреть элемент на вершине без удаления;
- `isEmpty()` — проверка на пустоту;
- `size()` — текущее количество элементов;
- реализация `Iterable<T>` позволяет обходить элементы в цикле `for-each`.

**Код для вставки в отчёт:** интерфейс `SimpleStack` целиком.

#### 2.2. Стек на массиве (`ArrayStack`)

Реализация `ArrayStack<T>` использует внутренний массив `Object[] data`, который при необходимости расширяется в два раза (динамический массив).

- Вершина стека соответствует последнему занятому элементу массива с индексом `size - 1`.
- Операция `push`:
  - при переполнении массива создаётся новый массив удвоенной длины;
  - элемент записывается в конец и `size` увеличивается.
- Операция `pop`:
  - проверяет пустоту, уменьшает `size`, возвращает элемент с вершины и обнуляет ячейку для помощи сборщику мусора.
- Итератор обходит стек **от вершины к основанию**, что удобно для визуального просмотра содержимого.

**Код для вставки в отчёт:** класс `ArrayStack` целиком, с комментариями.

#### 2.3. Стек на односвязном списке (`LinkedStack`)

Реализация `LinkedStack<T>` основана на внутреннем классе `Node<E>`, представляющем узел односвязного списка:

- каждый узел хранит значение `value` и ссылку `next` на следующий узел;
- поле `head` хранит ссылку на верхний элемент стека;
- `push` создаёт новый узел и делает его новой вершиной (`head = new Node<>(value, head)`), размер увеличивается;
- `pop` считывает значение из вершины, сдвигает `head` на следующий узел, уменьшает размер;
- итератор просто проходит по цепочке узлов, также от вершины к основанию.

**Код для вставки в отчёт:** класс `LinkedStack` целиком, с комментариями.

#### 2.4. Структура `Person`

Для демонстрации работы стека с объектами используется простой класс `Person` внутри `StackLab3`:

- поля: имя (`name`) и дата рождения (`birthDate` типа `LocalDate`);
- используется в тесте с выборкой людей по возрасту.

**Код для вставки в отчёт:** вложенный класс `Person` из `StackLab3`.

### 3. Описание тестов и экспериментов

Все тесты реализованы в классе `StackLab3` (файл `StackLab3.java`).  
Метод `main` последовательно вызывает несколько демонстрационных тестов.

#### 3.1. Тест 1 — работа со стеком целых чисел

Метод `testIntegers()`:

- создаёт два стека: `ArrayStack<Integer>` и `LinkedStack<Integer>`;
- генерирует 1000 случайных целых чисел в диапазоне [-1000; 1000];
- для каждого стека:
  - последовательно кладёт все числа в стек (`push`);
  - затем полностью извлекает элементы (`pop`), параллельно считая:
    - сумму,
    - минимальное и максимальное значение,
    - среднее значение.
- выводит на экран строку вида:  
  `ArrayStack: n=1000, sum=..., min=..., max=..., avg=...`

**Код для вставки:** методы `testIntegers()` и `runIntStackTest(...)`.

#### 3.2. Тест 2 — стек строк

Метод `testStrings()`:

- использует стек `ArrayStack<String>`;
- в стек заносится 10 строковых значений;
- содержимое стека выводится при помощи цикла `for-each` (итератор);
- затем элементы по одному извлекаются методом `pop`, и выводится их порядок.

Тест демонстрирует:

- работу операций `push` и `pop` на наглядном примере;
- порядок LIFO (последний добавленный выходит первым);
- использование собственного итератора.

**Код для вставки:** метод `testStrings()`.

#### 3.3. Тест 3 — работа со структурой `Person` и датами рождения

Метод `testPersonsByBirthDate()`:

- создаёт стек `LinkedStack<Person>`;
- генерирует 50 объектов `Person` с датами рождения в промежутке с 01.01.1980 по 01.01.2020 и кладёт их в стек;
- далее последовательно извлекает элементы из стека и для каждого человека:
  - вычисляет возраст на фиксированную дату (например, 01.01.2026);
  - считает, сколько людей младше 20 лет;
  - сколько людей старше 30 лет.

На выходе получается строка:  
`Всего людей: 50, младше 20: X, старше 30: Y`

Тест иллюстрирует:

- работу стека с объектами и полями разных типов;
- использование стека для фильтрации по условию без сохранения исходного порядка.

**Код для вставки:** метод `testPersonsByBirthDate()` и вспомогательный `randomDateBetween(...)`.

#### 3.4. Тест 4 — инвертирование последовательности с помощью двух стеков

Метод `testInversionWithStacks()`:

- создаёт два стека `original` и `inverted` (оба `ArrayStack<Integer>`);
- в `original` кладутся числа 1..10 (1 внизу, 10 на вершине);
- затем элементы перекладываются:  
  пока `original` не пуст, выполнить `inverted.push(original.pop())`;
- в результате порядок элементов разворачивается;
- выводится содержимое `inverted` — получаем последовательность в обратном порядке.

Тест показывает, как с помощью только операций `push`/`pop` можно инвертировать содержимое контейнера без прямого обращения к индексам.

**Код для вставки:** метод `testInversionWithStacks()`.

#### 3.5. Тест 5 — сравнение производительности реализаций

Метод `comparePerformance()`:

- генерирует массив из 10000 целых чисел;
- для `ArrayStack` и `LinkedStack` по очереди:
  - измеряет время (в наносекундах), затраченное на:
    - последовательное добавление всех элементов в стек (`push`);
    - полное извлечение элементов (`pop`);
  - переводит время в миллисекунды;
- выводит две строки вида:
  - `ArrayStack: push+pop 10000 элементов заняли X мс`
  - `LinkedStack: push+pop 10000 элементов заняли Y мс`

По результатам обычно видно, что стек на массиве работает немного быстрее за счёт лучшей локальности по памяти, а стек на списке имеет небольшие накладные расходы на создание узлов.

**Код для вставки:** методы `comparePerformance()` и `measurePushPop(...)`.

### 4. Результаты экспериментов

При запуске `StackLab3.main` в консоль выводится последовательность результатов по каждому тесту. Примерно:

- для целых чисел — одинаковые агрегаты (sum, min, max, avg) для обеих реализаций, что подтверждает корректность работы;
- для строк — порядок вывода с помощью итератора и порядок извлечения через `pop` демонстрируют поведение LIFO;
- для людей — подсчитывается разумное количество людей в возрастных категориях «младше 20» и «старше 30»;
- при инверсии — на экран выводится развернутая последовательность чисел;
- при сравнении производительности — стек на массиве показывает немного меньшее время по сравнению со стеком на списке на коллекции из 10000 элементов.

Конкретные числовые значения могут отличаться при другом seed генератора, но качественная картина сохраняется.

### 5. Выводы

По результатам лабораторной работы реализованы две корректные версии стека — на динамическом массиве и на односвязном списке — которые поддерживают базовые операции push, pop и peek, а также проверку на пустоту, подсчёт количества элементов и обход содержимого через собственный итератор. Проведённые тесты на целых числах, строках и объектах с датой рождения подтвердили правильность работы обеих реализаций и соответствие поведению LIFO, а также показали возможность использовать стек для практических задач, таких как вычисление агрегатных характеристик и инвертирование последовательности с помощью только операций вставки и изъятия. Экспериментальное сравнение на коллекции из 10000 элементов продемонстрировало, что реализация на массиве обычно выполняет операции немного быстрее благодаря компактному размещению данных и лучшей локальности памяти, тогда как реализация на списке обеспечивает гибкое расширение без копирования массива, но имеет накладные расходы на создание узлов. В целом обе реализации пригодны для использования, а выбор между ними зависит от требований к скорости, памяти и характеру нагрузки.
